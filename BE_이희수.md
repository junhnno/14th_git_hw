Git은 내 컴퓨터에서 버전 관리를 하는 프로그램 / GITHUB는 GIT을 온라인으로 공유하는 플랫폼 => 즉 Git과 GITHUB는 같이 써야 의미가 있다.
git init 명령어를 이용해 깃허브와 연동
add, commit, push - 파일의 수정사항을 추가하고 Git 이력에 기록하고 브랜치에 올리기
pull이 매우매우 중요한데, 수정사항을 반영하기 위해 항상 push 직전 pull을 꼭 해주기!!
충돌은 같은 파일의 같은 부분을 서로 다르게 수정한 경우, 한 명은 수정하고 한 명은 그것을 삭제한 경우, pull을 하지 않고 push했을 때 발생
해결 방법으로는 pr에서 해결하는 경우, pull을 받아 vscode로 해결하는 경우가 있다.
